Patrón de Diseño

Memento

Guarda el estado sin romper el encapsulamiento

01 El Problema

¿Cómo guardar estado sin romper el encapsulamiento?

Un editor de texto, un formulario largo o un juego permiten modificar el estado de un objeto poco a poco. En algún momento el usuario se equivoca y quiere volver atrás (Ctrl+Z).

El problema es cómo guardar ese estado anterior sin romper el encapsulamiento del objeto. Si expones todos los campos internos para que otra clase pueda "guardarlos y restaurarlos", cualquiera puede leer y modificar el estado interno libremente — lo cual viola un principio fundamental de la OOP.

"Necesitas guardar una 'foto' del estado sin que quien la guarda sepa qué hay dentro de la foto."
02 ; La Solución

Tres roles, una responsabilidad cada uno

Memento separa tres actores con responsabilidades claras y bien delimitadas:

  • Originator El objeto cuyo estado cambia con el tiempo (editor, documento, personaje del juego). Sabe crear un memento de sí mismo y restaurarse a partir de uno.
  • Memento Objeto inmutable que guarda una copia del estado del Originator. Solo el Originator puede leer su contenido; hacia afuera es una caja negra.
  • Caretaker Guarda la lista de mementos (historial de undo/redo) pero nunca mira ni modifica su contenido. Solo pide "guárdame este" y "dame el anterior". 
 Originator           Memento          Caretaker
+-----------+     +------------+    +-------------+
| -state    |---->| -state(prv)|<---| -history[]  |
| +save()   |     +------------+    | +backup()   |
| +restore()|                       | +undo()     |
+-----------+                       +-------------+
03 ; Código Mínimo

Implementación en PHP

Código Incorrecto Rompe el encapsulamiento
class Memento {
  public string $state;

  public function __construct(string $state) {
    $this->state = $state;
  }
}

class History {
  public function undo(Memento $m): void {
    $rawState = $m->state;
    $m->state = "estado alterado";
  }
}
✓ Código Correcto Encapsulamiento respetado
class Memento {
  public function __construct(
    private readonly string $state
  ) {}

  public function getState(): string {
    return $this->state;
  }
}

class Editor {
  private string $content = "";

  public function write(string $text): void {
    $this->content .= $text;
  }
  public function save(): Memento {
    return new Memento($this->content);
  }
  public function restore(Memento $m): void {
    $this->content = $m->getState();
  }
}

class History {
  private array $stack = [];

  public function backup(Editor $e): void {
    $this->stack[] = $e->save();
  }
  public function undo(Editor $e): void {
    $m = array_pop($this->stack);
    if ($m) $e->restore($m);
  }
}
04 Cuándo Aplicarlo

Úsalo bien o evítalo

✔ Sí aplica
  • Necesitas undo/redo real — editores, dibujo, juegos.
  • Quieres puntos de restauración en una transacción.
  • El estado es relativamente pequeño o usas snapshots parciales.
✘ No aplica
  • El objeto no cambia de estado o nunca se necesita "volver atrás".
  • El estado es enorme — mejor usar Command con deltas.
  • Re-consultar la fuente de datos (BD) es suficiente.
05 Ejemplos Reales

Memento en el mundo real (verificables)

☕

Java Swing — javax.swing.undo

El paquete estándar de Java implementa esta idea directamente para el soporte de undo/redo en componentes Swing. La interfaz StateEditable permite que un objeto exponga su estado mediante storeState() y restoreState(). La clase StateEdit actúa como el "memento" que guarda ese estado para poder revertirlo.

Es documentación oficial de Java SE, fácil de citar y verificar en docs.oracle.com bajo el paquete javax.swing.undo.

Implementación formal
🌿

Git — Snapshots de repositorio

Los sistemas de control de versiones como Git guardan snapshots completos del estado del repositorio en cada commit. Conceptualmente, cada commit es un memento: preserva una "foto" del estado de todos los archivos en ese momento, y el historial de commits actúa como el Caretaker que guarda la colección de esos snapshots.

Git no usa las clases literales del patrón GoF, pero la analogía es directa e ilustrativa del principio fundamental de Memento.
